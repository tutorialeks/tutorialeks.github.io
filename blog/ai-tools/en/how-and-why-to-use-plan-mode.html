Mastering Cursor: Using Plan Mode and Act Mode to Write Better Code

How AI-powered planning and acting reduce refactoring and improve development workflow

What is Cursor?

Cursor is an AI-first integrated development environment (IDE) that leverages advanced AI agents to assist programmers in writing code faster and smarter. Beyond simply generating code, Cursor introduces modes that optimize when and how coding happens to create cleaner, more maintainable projects.

The Two AI Modes: Plan and Act

Cursor uses two complementary modes for AI agents: Plan mode and Act mode.

  • Plan mode: In this mode, the AI agent discusses, analyzes, and devises a detailed plan before generating code. It thinks through edge cases, dependencies, and task structure to minimize revisions later.
  • Act mode: After planning, the AI switches to Act mode to implement the plan by writing or editing code directly based on the agreed approach.

Why Plan Before You Act?

Previously, AI coding assistants often jumped straight into generating code, which led to frequent refactoring when overlooked requirements or architectural decisions became apparent late.

By discussing the problem first in Plan mode, developers can outline critical details early, allowing the AI to generate higher quality code with fewer errors or omissions.

This approach saves time and reduces bugs caused by rushed coding and rework.

Example: Handling Multilingual URLs with Cursor

For example, in one of my projects, the AI initially did not plan distinct URLs for Bulgarian and English forms, resulting in overlapping routes and confusion.

Using Plan mode, I asked the AI agent to revisit the routing strategy and explicitly plan different URLs for each language, explaining the reason and impact.

Once the plan was agreed upon, switching to Act mode let the AI efficiently implement the correct routing logic with minimal refactoring afterward.

How to Use Plan Mode and Act Mode in Cursor

  1. Activate Plan mode: Start your coding session by enabling Plan mode in Cursor. Input your problem statement or feature request clearly with any context or constraints.
  2. Engage in AI discussion: Review the AI’s proposed plan, ask questions, request clarifications, and suggest adjustments until the plan meets your expectations.
  3. Switch to Act mode: Once the plan is finalized, switch Cursor to Act mode where the AI writes the code according to the plan.
  4. Review and iterate if needed: Validate the generated code and only request changes if the implementation strays from the agreed plan, which should now be rare.

By leveraging Cursor’s Plan and Act modes, developers can collaborate with AI agents effectively, allowing more thoughtful, structured coding sessions that minimize time-consuming refactoring and improve overall code quality.
